# 攻擊向量目錄

> 系統化的攻擊面向，確保「攻得全面」——不是隨便攻，而是按面向逐項攻。

依命題的 type 從下表選相關類別。每類有幾個攻擊模式 + steel-manned（最強化）範例。

## 通用面向（決策/架構/計畫/策略/投資）

### 假設（ASSUMPTIONS）— 高風險，失效前常隱形
- **隱藏假設揭露**：揪出沒明說、卻撐著整個命題的信念。例：「你假設使用者願意多付 20%，但沒有定價研究支撐，市場上類似功能都免費。」
- **承重挑戰**：命題成敗繫於某假設，但信心不足。例：「整個 ROI 押在 30% 採用率，業界基準是 10-15%；若 15%，NPV 轉負。」
- **假設堆疊**：成功要 A 且 B 且 C 都成立，各 70%，合起來只剩 34%，且無 fallback。
- **時效衰減**：假設依賴「現況持續」，但條件會變，有沒有 hedge？

### 依賴（DEPENDENCIES）
- 外部依賴消失（上游 API/服務/marketplace 下線、套件棄更）。
- 內部依賴循環、單點故障、版本鎖死。

### 邊界案例（EDGE_CASES）
- 空輸入/極大輸入/同名碰撞/併發/跨平台差異（如 Windows 路徑、權限）。

### 擴展性（SCALABILITY）
- 量大/量小時崩潰；成長後維護成本爆炸。

### 維運（OPERATIONAL）
- 日常執行卡點、人為流程斷鏈、需要某人手動的步驟容易被忘。

### 競爭 / 市場 / 經濟（COMPETITIVE / ECONOMIC）
- 競爭者反制、市場條件變化、財務可行性、現金流。

### 時機（TEMPORAL）
- 時程無 buffer、依賴的事件延遲、視窗稍縱即逝。

### 人 / 組織（ORGANIZATIONAL）
- 需要人改變習慣（歷史變革 70% 失敗率）、文化阻力、權責不清。

## 程式碼面向（code）— 本專案實戰

- **崩潰**：null ref、無效輸入、資源耗盡、競態、溢位。要可重現。
- **資安/注入**：injection、path traversal、控制字元注入、資料外洩、不安全反序列化。要 PoC。
- **跨平台**：Windows cmd.exe 不認 Unix 語法（`2>/dev/null`、`&&`）、檔案 symlink 需權限、CRLF。（本 session 多次踩到。）
- **錯誤處理一致性**：JSON.parse 沒包 try/catch、fail-safe 把無法確認當成 0（如 rev-list NaN 當已 push）。
- **邏輯/狀態**：dirty 狀態機沒閉合、entry 缺漏靜默降級、原子寫缺失。
- **路徑逃逸**：name 含 `../` 逃出目錄；要白名單驗證。

## plugin/skill/agent 配置面向（config）— 本專案實戰

- **觸發詞衝突/失靈**：description 觸發詞與別的 skill 撞、或全英文使用者中文觸發不到。
- **權限過寬**：plugin 拿到不需要的工具、hook 能跑任意指令。
- **路徑逃逸 / 敏感外洩**：registry 存絕對路徑進 git、token 沒被 .gitignore 排除、adoptedFrom 沒脫敏。
- **互動指令邊界**：Claude 代執行了不該代執行的 `/plugin`；或文件叫使用者貼根本不存在的指令（如 `/plugin update`）。
- **覆蓋風險**：狀態存 plugin 內、被更新覆蓋（vs 該存家目錄）。

## 文件一致性面向（docs）— 本專案實戰

- **文件聲稱 vs 實際**：教學說有某功能/某 hook，實際不存在（吹牛）。
- **殘留過時**：移除某指令/欄位後，文件還提它（如殘留 `:new`、`externalCandidates`）。
- **版號/清單不符**：plugin.json bump 了，教學版號沒同步；scripts/plugin 清單與實際不符。
- **斷鏈**：目錄錨點指向不存在的章節。
- **指令可貼性**：文件示範 `/plugin uninstall && install`（slash 不能 && 串接）。

## 嚴重度基準

| 嚴重度 | 準則 | 例 |
|------|------|----|
| CRITICAL | 承重 + 信心 < 50% | 「市場存在」零驗證、會丟資料的狀態機漏洞 |
| HIGH | 承重 或 信心 < 50% | 時程無 buffer、跨磁碟整個功能壞 |
| MEDIUM | 影響效率非可行性 | UX 不一致、潛在 robustness（當前不觸發） |
| LOW | 錯了可補救 | 文件措辭、註解殘留 |
