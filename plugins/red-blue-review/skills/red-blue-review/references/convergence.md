# 收斂準則

> 知道何時該停——攻夠了沒。

## 三種收斂模式

| 模式 | 停止條件 | 適用 |
|------|---------|------|
| `no_new_critical`（預設） | 這輪產生 0 個新 CRITICAL/HIGH 攻擊 | 標準審查 |
| `all_addressed` | 沒有 ACCEPT 殘留（全部 REFUTE/MITIGATE/HARDEN） | 高風險決策 |
| `round_limit` | 到上限輪數（預設 3，1-5） | 限時審查 |

## 「新攻擊」怎麼算（去重）

每輪數新攻擊前，先剔除重複/變體：

- **重複**：跟前面攻擊一模一樣。
- **變體**：同一個底層顧慮換個說法、或範圍稍微放大。例：
  - 「團隊缺 K8s 經驗」vs「團隊缺分散式系統經驗」→ 同顧慮、變體，**不算新**。
- **真新**：針對「明顯不同的弱點」，才算新攻擊。

只有「真新」的 CRITICAL/HIGH 才讓 `no_new_critical` 繼續下一輪。

## 本專案實戰：loop-until-dry

對「未知數量」的弱點挖掘（不確定還有沒有），可用「連續 K 輪沒新發現才停」：
- 跑一輪紅藍 → 有新確認弱點 → 重置計數、再跑
- 連續 2 輪 0 新發現 → 收斂
- 去重要對「所有已見過的」比，不是只對「已確認的」——否則被藍方剔除的假陽性會每輪重現、永不收斂。

## 收斂後輸出 go/no-go

- 所有 CRITICAL/HIGH 都 REFUTE（證實是假）/HARDEN/MITIGATE → **GO**
- 還有未解的 CRITICAL → **BLOCK**
- ACCEPT 的殘餘風險要明列，讓使用者知道「帶著什麼風險上路」。
