# 收斂準則

> 知道何時該停——攻夠了沒。

## 收斂前置：面向覆蓋閘

任何準則宣告收斂前，必須先過**覆蓋閘**：該標的 type 在 SKILL.md type 表所列的「預設攻擊面向」**每一項都至少被認真攻過一輪**（steel-manned，非一句帶過）。記一份 covered 面向 checklist（面向名 → 已攻/未攻 + 該輪最強攻擊摘要）。存在未攻面向時不得宣告收斂，須先補攻——否則退化成只攻淺面向就 GO 的「確認劇場」。

## 核心收斂條件：迴圈到攻不破（唯一輪 0 新即停）

**兩模式共用、預設行為**：紅攻 → 藍守 →（修）→ 再跑一輪……**直到某一輪紅方產生 0 個新的真弱點**就收斂。

- 「新的真弱點」= 藍方驗證為真（非假陽性）+ 去重後非變體。
- **一輪 0 新即停**（不需連續多輪確認）——前提是已過下方「面向覆蓋閘」（確保不是只攻淺面向就 0 新）。
- **修復複驗前置**：宣告收斂前，歷輪「宣稱已修」的 finding 必須**全數通過修復複驗**（實讀改後檔案驗落地 + 原攻擊路徑再打一次；見 SKILL 第四步）。「修了」只認驗過的產出、不認修方自述——存在複驗不過項 → 不得收斂，退回重修。
- **硬上限**：迴圈受 round 上限保護（預設 5 輪），達上限即強制停並回報殘餘弱點，避免極端情況攻不完。

### 可調參數（覆寫核心行為時用）

| 參數 | 作用 | 預設 |
|------|------|------|
| `round_cap` | 迴圈硬上限輪數 | 5（可 1-10） |
| `dry_rounds` | 要連續幾輪 0 新才停（提高嚴謹度用） | 1（即唯一輪 0 新即停） |
| `all_addressed` | 額外要求：每個 CRITICAL/HIGH 須 REFUTE/HARDEN/MITIGATE，或 ACCEPT 且明列殘餘風險才停（高風險決策用） | 關 |

> `all_addressed` 不要求「零 ACCEPT」——真實且不可消除的殘餘風險，只要明列在 go/no-go 即視為 addressed，允許收斂並標「帶風險上路」。否則高風險決策（最常有不可消除殘餘風險）會死鎖。

## 「新攻擊」怎麼算（去重）

每輪數新攻擊前，先剔除重複/變體：

- **基準**：去重一律對「**所有已提出過的攻擊（含已被藍方剔除的假陽性）**」比，**不是只對「已確認的」**——否則被剔除的假陽性會每輪被紅方重新提出、計為新弱點，使「0 新即停」永遠無法滿足、迴圈不收斂。
- **只看底層顧慮（root concern）**：與嚴重度/措辭無關。
- **重複**：跟前面攻擊一模一樣。
- **變體**：同一個底層顧慮換個說法、或範圍稍微放大。例：「團隊缺 K8s 經驗」vs「團隊缺分散式系統經驗」→ 同顧慮、變體，**不算新**。可操作判準（heuristic）：兩攻擊的修法是否相同 → 相同則視為同顧慮。
- **真新**：針對「明顯不同的弱點」，才算新攻擊。

只有「真新」的弱點才讓迴圈繼續下一輪；一輪內全是重複/變體/假陽性 = 0 新 = 該輪收斂。

## 機械閘：讓「要不要再跑一輪」由資料判定，不靠模型自律

> **這三道是收斂能不能擋住「紅方鑽牛角尖硬抓錯誤、硬湊 LOW 讓迴圈不停」的關鍵。** 它們是**機械判定**（照 finding 的欄位值機械過濾），不是散文勸導——對話內手動跑時逐 finding 照表執行，接 Workflow 時由 `loop-runner.md` 的腳本控制流強制。三道全過濾完，才數「本輪新真弱點」。

| # | 閘 | 規則（機械） | 擋掉什麼 |
|---|----|------------|---------|
| 1 | **真偽閘** | 藍方 `is_real:false` 的 finding → **直接丟棄，不進任何計數** | 紅方的假陽性（false finding），不讓它充數 |
| 2 | **嚴重度閘** | `corrected_severity < MEDIUM`（含 LOW）→ **不計入「新弱點」**，迴圈視同該項不存在 | 「為了續圈硬找 LOW 級雞毛蒜皮」——湊 LOW 不會讓 dry 歸零 |
| 3 | **去重閘** | 去重對「**所有已提過的攻擊（含已被剔除的假陽性）**」比 root concern → 變體/重複不算新 | 被打回的假 finding 換個說法重刷一輪、騙過收斂 |

> **閘3 豁免（修復複驗不過，MANDATORY）**：某 root_concern 對應的修復若**複驗不過**（宣稱修了、實讀改後檔案發現沒落地或原攻擊仍打得穿），該 root_concern **必須從 seen 移除**——下一輪紅方重提**照計新弱點**，不受去重閘吞掉。去重閘只保護「已複驗確認修好」與「判假」的項，**不保護「宣稱修了」**；不做此豁免，「沒修好」會被當變體吞掉、永遠隱形，收斂形同虛設。
>
> ⚠️ **閘3 的已知取捨（誠實揭露）**：把假陽性也記入「提過的」是為擋換皮重刷，代價是「某 root_concern 首輪判假/LOW、次輪同字串卻真 ≥MEDIUM」會被誤殺。緩解：root_concern 命名要具體（降低不同問題撞同字串）。高風險命題要完全避免此誤殺的改法見 `loop-runner.md` 鐵律段。預設選「防永不收斂」優先。
>
> **但誤殺不可「靜默」——強制留痕（MANDATORY）**：當一個 finding 因閘3被去重、但它本輪的 `is_real:true 且 corrected_severity ≥ MEDIUM`（即「本輪其實是真且夠嚴重，只因 root_concern 撞到過去被剔除/降級的項而被吞」），**不計入迴圈計數，但必須記進報告的「⚠️ 去重吞掉的疑似真弱點」清單**（含 root_concern、本輪 severity、為何被判為變體）。這把「靜默漏掉」變成「使用者看得到的待確認項」——收斂可以繼續，但漏掉的東西不再隱形。第五步產出必須帶上此清單（無則明寫「無」）。

**計數公式（本輪新真弱點數）**：

```
新真弱點 = findings
  .filter(f => f.verdict.is_real === true)              // 閘1：真偽
  .filter(f => sev(f) >= MEDIUM)                         // 閘2：嚴重度（sev 取 corrected_severity，無則 fallback 紅方 severity）
  .filter(f => !seen.has(rootConcern(f)))               // 閘3：去重（對所有提過的）
  .length
// 本數 === 0（且覆蓋閘已過）→ 該輪 dry，收斂
```

> 三道閘合起來的效果：**紅方再怎麼想讓迴圈繼續，能讓 `dry` 歸零的唯一方法是提出「藍驗為真、≥MEDIUM、且前所未見」的弱點**——也就是真的還有真問題。湊假的、湊小的、換皮的，全被閘擋在計數外。
>
> **兩種保證等級（別混淆）**：接 Workflow 跑時，上面的 filter 是**真的 JS 程式在算**（機械運算，模型繞不過）。對話內跑時，`is_real`/`corrected_severity` 由模型自填，這三道閘是模型「按規則自我裁決」——是**自律強化版**，非硬保證。對話內模式請把上面的 filter 當「必須誠實套用的自我檢查」，而非「有外部程式幫你擋」。

## 迴圈跑法

- 跑一輪紅藍（→ 實作模式立即修；分析模式首輪停給確認清單，確認後修）→ 有新確認弱點 → 再跑下一輪
- 某輪 0 新真弱點（且覆蓋閘已過）→ 收斂
- 達 `round_cap`（預設 5）→ 強制停並回報殘餘弱點
- 提高嚴謹度可設 `dry_rounds > 1`（需連續多輪 0 新才停）

## 收斂後輸出 go/no-go

- **攻擊覆蓋**：依 attack-catalog 對選定 type 的預設面向逐項標三態——「已攻+驗證無弱點」/「已攻+有 finding」/「未攻到」。
- 所有 CRITICAL/HIGH 都 REFUTE（證實是假）/HARDEN/MITIGATE → **GO**
- 還有未解的 CRITICAL/HIGH → **BLOCK**
- 任一**承重**面向落在「未攻到」→ GO 須降級為「GO（覆蓋不全，N 個面向未攻：…）」，高風險模式下視同未收斂。
- ACCEPT 的殘餘風險要明列，讓使用者知道「帶著什麼風險上路」。
