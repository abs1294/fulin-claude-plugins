# 收斂準則

> 知道何時該停——攻夠了沒。

## 收斂前置：面向覆蓋閘

任何準則宣告收斂前，必須先過**覆蓋閘**：該標的 type 在 SKILL.md type 表所列的「預設攻擊面向」**每一項都至少被認真攻過一輪**（steel-manned，非一句帶過）。記一份 covered 面向 checklist（面向名 → 已攻/未攻 + 該輪最強攻擊摘要）。存在未攻面向時不得宣告收斂，須先補攻——否則退化成只攻淺面向就 GO 的「確認劇場」。

## 四種收斂準則

| 準則 | 停止條件 | 適用 |
|------|---------|------|
| `no_new_critical`（預設） | **覆蓋閘通過** 且這輪 0 個新 CRITICAL/HIGH | 標準審查 |
| `all_addressed` | 每個 CRITICAL/HIGH 都已 REFUTE/HARDEN/MITIGATE，或 ACCEPT 且附明確殘餘風險記錄（列入 go/no-go 殘餘風險清單） | 高風險決策 |
| `round_limit` | 到上限輪數（預設 3，1-5） | 限時審查 |
| `loop-until-dry` | 連續 K 輪（預設 2）0 新發現；仍受 round_limit 硬上限保護，避免無限循環 | 範圍未知/全面稽核（如 config/docs） |

> `all_addressed` 不要求「零 ACCEPT」——一個真實且不可消除的殘餘風險，只要明列在 go/no-go 即視為 addressed，允許收斂並標「帶風險上路」。否則高風險決策（最常有不可消除殘餘風險）會死鎖。

## 「新攻擊」怎麼算（去重）

每輪數新攻擊前，先剔除重複/變體：

- **基準**：去重一律對「**所有已提出過的攻擊（含已被藍方剔除的假陽性）**」比，**不是只對「已確認的」**——否則被剔除的假陽性會每輪被紅方重新提出、計為新 CRITICAL/HIGH，使收斂永遠無法滿足。此規則適用全部四種準則。
- **只看底層顧慮（root concern）**：與嚴重度/措辭無關。
- **重複**：跟前面攻擊一模一樣。
- **變體**：同一個底層顧慮換個說法、或範圍稍微放大。例：「團隊缺 K8s 經驗」vs「團隊缺分散式系統經驗」→ 同顧慮、變體，**不算新**。可操作判準（heuristic）：兩攻擊的修法是否相同 → 相同則視為同顧慮。
- **真新**：針對「明顯不同的弱點」，才算新攻擊。

只有「真新」的 CRITICAL/HIGH 才讓 `no_new_critical` / `loop-until-dry` 繼續下一輪。

## loop-until-dry 跑法

對「未知數量」的弱點挖掘（不確定還有沒有）：
- 跑一輪紅藍 → 有新確認弱點 → 重置計數、再跑
- 連續 K 輪（預設 2）0 新發現 → 收斂（仍受 round_limit 硬上限：達上限即強制停）

## 收斂後輸出 go/no-go

- **攻擊覆蓋**：依 attack-catalog 對選定 type 的預設面向逐項標三態——「已攻+驗證無弱點」/「已攻+有 finding」/「未攻到」。
- 所有 CRITICAL/HIGH 都 REFUTE（證實是假）/HARDEN/MITIGATE → **GO**
- 還有未解的 CRITICAL/HIGH → **BLOCK**
- 任一**承重**面向落在「未攻到」→ GO 須降級為「GO（覆蓋不全，N 個面向未攻：…）」，高風險模式下視同未收斂。
- ACCEPT 的殘餘風險要明列，讓使用者知道「帶著什麼風險上路」。
