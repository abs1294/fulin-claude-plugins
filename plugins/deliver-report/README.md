# deliver-report

把本次 session 的**交付產出**寫成一段口語、務實、逐檔導讀的**交付訊息**——固定 fulin 慣用的格式與語氣，內容依當次產出而定。

## 解決的痛

每次要把一批檔案（分析報告 + 明細 + 原始資料 + 實作/設定檔）交給收件人時，都要手寫一段導讀訊息。格式與語氣有既定偏好，但每次重寫很煩、還容易漏掉版本一致性檢查。這個 skill 把那個「滿意的格式」固定下來，只換內容。

## 觸發

說「寫交付訊息」、「交付這批檔案」、「產出交付信」、「幫我寫給客戶/工程端的說明」、「整理成交付訊息」、「deliver」即觸發。

## 產出長這樣（骨架）

```
Hi <收件人>，

<這次的 XX 分析／產出，共 N 份檔案：>
<檔名> — <一句白話：這份是什麼、去哪查什麼>
...
<背景：資料從哪來、範圍（時間/筆數）>，<帶數字的重點結論>
<（有前版才寫）跟上次的差異＋原因>
<導讀動線：先看哪份抓全貌 → 要查細節翻哪份>，有問題再跟我說~

謝謝！
```

## 特點

- **格式與語氣固定，內容依產出而定**：這是使用者要固定的核心。
- **技術詞翻人話**：專有名詞只保留在「給工程端那一檔」的說明裡，其餘白話。
- **收件人留空**：預設 `Hi <收件人>，` 不預填人名。
- **寫前順手整理**：列實際檔案、檢查版本一致性、依邏輯排序（彙總→明細→原始資料→實作/設定），但不擅自改檔。

前置依賴：無。
