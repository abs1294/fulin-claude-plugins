# Changelog

本檔記錄 deliver-report 的版本變更，格式依 [Keep a Changelog](https://keepachangelog.com/)。

## [0.2.0] - 2026-07-06
### Changed
- 新增第三型「多受眾分流型」（▌ 按受眾切區塊：該讀哪幾份＋閱讀順序＋一行重點＋範圍邊界聲明）＋選型規則表（判準=收件人結構非檔案數）；語氣規則加「分流型術語分級按區塊不按檔」；收錄碳排模組信為範例三 few-shot 並擴充提煉要點（同檔可跨區塊、重點行=濃縮先修、邊界先講）

## [0.1.2] - 2026-07-03
### Fixed
- 加第二個展開敘事型範本(OppTrack)：開場回應委託、逐檔展開子項、結論改帶判斷的敘事(核心/風險最高優先/零流量可移除)、誠實講做不到；骨架改為可有機展開非填空格；新增硬規則：對外說明嚴禁任何內部術語與流程字眼(紅藍對抗/QA/code-review/adopt/bump/skill/plugin/session/AI…)

## [0.1.1] - 2026-07-03
### Fixed
- 收件人沒指定改輸出乾淨的 Hi，（不再留 <收件人> 佔位符，忘改會難看）；強化語氣規則：預設讀者不懂技術，術語只留給工程端那一檔、其餘白話，少用英文縮寫

## [0.1.0] - 2026-07-03
### Added
- 首版：交付訊息產生器。把本次 session 的交付產出寫成一段口語、務實、逐檔導讀的交付訊息，固定 fulin 慣用格式與語氣（Hi 稱呼→一句話交代這批是什麼→逐檔白話清單→背景範圍與帶數字的重點結論→有前版才寫的差異＋原因→導讀動線→輕鬆收尾）。
- 語氣規則明列：口語務實、技術詞翻人話（專有名詞只留在給工程端那一檔）、結論先講帶數字、變動要給原因不只報數字、輕鬆收尾。
- 收件人不預填人名（使用者要求）。
- 寫訊息前順手整理交付清單：列實際檔案、檢查版本一致性、依「彙總→明細→原始資料→實作/設定」邏輯排序，不擅自改檔。
- 內建 few-shot 範例（AWS WAF 交付），作為語氣與密度的對齊基準。
