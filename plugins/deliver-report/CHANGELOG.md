# Changelog

本檔記錄 deliver-report 的版本變更，格式依 [Keep a Changelog](https://keepachangelog.com/)。

## [0.1.0] - 2026-07-03
### Added
- 首版：交付訊息產生器。把本次 session 的交付產出寫成一段口語、務實、逐檔導讀的交付訊息，固定 fulin 慣用格式與語氣（Hi 稱呼→一句話交代這批是什麼→逐檔白話清單→背景範圍與帶數字的重點結論→有前版才寫的差異＋原因→導讀動線→輕鬆收尾）。
- 語氣規則明列：口語務實、技術詞翻人話（專有名詞只留在給工程端那一檔）、結論先講帶數字、變動要給原因不只報數字、輕鬆收尾。
- 收件人預設留空佔位符 `Hi <收件人>，`，不預填人名（使用者要求）。
- 寫訊息前順手整理交付清單：列實際檔案、檢查版本一致性、依「彙總→明細→原始資料→實作/設定」邏輯排序，不擅自改檔。
- 內建 few-shot 範例（AWS WAF 交付），作為語氣與密度的對齊基準。
